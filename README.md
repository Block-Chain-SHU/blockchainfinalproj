# 计算机专业人才培养与成果追踪系统

基于 Hyperledger Fabric、Go、Vue 的多角色链上追踪系统示例项目。  
本项目用于展示“学生培养信息 -> 分阶段上链 -> 追踪查询”的完整流程。

## 项目定位

- 面向计算机专业场景的培养过程追踪
- 支持多角色分阶段录入
- 支持链上不可篡改记录与历史查询

## 角色设计

- 学生
- 课程/实验室
- 项目实践/竞赛导师
- 就业单位/研究生去向
- 学院/企业/本人查询（只读）

## 技术栈

- 区块链网络：Hyperledger Fabric
- 后端：Go + Gin
- 前端：Vue2 + Element UI
- 部署方式：Docker + Shell 脚本

## 目录结构

- `blockchain/`：Fabric 网络与链码
- `application/backend/`：后端接口服务
- `application/web/`：前端页面与交互

## 本地运行（开发模式）

1. 启动区块链网络（Linux/WSL/Git Bash 环境）  
2. 启动后端服务：`application/backend`  
3. 启动前端服务：`application/web`  
4. 浏览器访问前端地址进行登录、录入、查询验证

## 注意事项

- 前端与后端地址请按你的运行环境配置（本机/虚拟机/云服务器）
- 若端口被占用，请先释放端口再启动服务
- 首次启动区块链网络时，拉取镜像耗时较长属于正常现象
